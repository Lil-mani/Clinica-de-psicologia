### Estrutura Detalhada das Sprints com Reuniões e Distribuição de Tarefas

#### Sprint 1: Reunião de Planejamento

##### Objetivo:
Definir tarefas iniciais focadas em CRUD básico, montagem da landing page, familiarização com Breeze, e estruturação do projeto.

##### Tarefas:

1. *CRUD Básico do Administrador*
   - *Luiz (Banco de Dados):*
     - Criar tabelas no banco de dados para psicólogo, cliente e secretária.
     - Implementar operações CRUD no back-end.
   - *Lucas (Testes):*
     - Testar a funcionalidade CRUD para cada entidade.
   - *Amanda (Front-end):*
     - Criar formulários e interfaces para CRUD de psicólogo, cliente e secretária.

2. *Montagem da Landing Page*
   - *Amanda (Front-end):*
     - Implementar seção "Sobre Nós".
     - Adicionar depoimentos.
     - Criar área para contatos.
     - Montar o formulário de contato inicial.
   - *Lucas (Testes):*
     - Testar funcionalidades e layout da landing page.

3. *Familiarização com Breeze*
   - *Amanda (Front-end):*
     - Implementar login e autenticação na landing page usando Breeze.
   - *Lucas (Testes):*
     - Testar funcionalidade de login e autenticação.

4. *Estruturação do Projeto*
   - *Todos:*
     - Definir e estruturar as páginas iniciais do projeto.
   - *Lucas (Testes):*
     - Verificar o funcionamento das rotas de cada página.

#### Sprint 2: Reunião de Planejamento

##### Objetivo:
Implementar cadastro, telas de usuários, funções do banco de dados, e back-end de login.

##### Tarefas:

1. *Cadastro utilizando Breeze e API de CEP*
   - *Amanda (Front-end):*
     - Desenvolver a interface de cadastro utilizando Breeze.
   - *Luiz (Banco de Dados):*
     - Integrar a API de CEP e armazenar os dados no banco.
   - *Lucas (Testes):*
     - Testar a funcionalidade de cadastro e integração da API de CEP.

2. *Implementação das Telas de Usuários*
   - *Amanda (Front-end):*
     - Criar páginas de histórico, agendamentos e agendamentos atuais.
   - *Lucas (Testes):*
     - Testar a funcionalidade e navegação entre as telas de usuários.

3. *Implementação de Funções do Banco de Dados*
   - *Luiz (Banco de Dados):*
     - Desenvolver funções para receber e armazenar informações do front-end.
   - *Lucas (Testes):*
     - Validar a correta recepção e armazenamento dos dados no banco.

4. *Implementação do Back-End de Login*
   - *Luiz (Banco de Dados):*
     - Desenvolver o back-end para login de usuário, secretária e psicólogo.
   - *Amanda (Front-end):*
     - Criar interfaces de login.
   - *Lucas (Testes):*
     - Testar a funcionalidade de login para todos os tipos de usuários.
